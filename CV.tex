\textbf{医疗 Agent 系统} \hfill 2026.02 -- 2026.04 \titlebreak
传统单 Agent 医疗问答系统存在\textbf{响应效率低、复杂问题分析不全面、多轮对话上下文理解能力弱}等问题。面对多症状、多维度的医疗咨询时，回答准确性与稳定性不足；同时缺乏长效记忆机制，无法连贯理解用户意图，难以满足实际医疗场景下高效、可靠、安全的智能问答需求。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{Skill-Agent两层架构}：底层构建7个原子Skills（知识检索、风险评估、症状分析、生活方式建议、ICD-10编码、临床指南、深度研究）供Agent使用，上层3个专业Agent（健康咨询、症状诊断、医学研究）根据任务需求自主选择调用，Skills和Agent完全解耦；
    \item \textbf{Agent loop执行机制}：基于ReAct框架实现Think-Act-Observe循环，LLM自主决策工具调用并观察结果，设置硬性限制最多调用2次工具防止过度调用，单Agent平均响应时间控制在15秒以内；
    \item \textbf{Agent Swarm}：智能路由判断问题复杂度（70\%单Agent快速通道/30\%Swarm协作模式），复杂问题自动分解任务并分配给多个Agent并行执行，Agent间通过共享上下文间接共享任务状态和执行结果；
    \item \textbf{双层记忆机制}：短期记忆管理会话级对话历史（单例模式确保所有Agent共享会话上下文，集成熵管理器自动去重压缩消息35\%，加载最近5轮对话），长期记忆通过Mem0云服务向量化存储会话总结支持跨会话相似案例检索；
    \item \textbf{Harness Engineering约束系统}:通过YAML文件显式定义Agent能力边界并运行时自动验证，实现输出自动修复机制（缺少免责声明自动添加、检测到高危症状自动添加就医警告）；
\end{itemize}
\textbf{项目成果：}

\textbf{MXWbot — 多通道智能 Agent 对话框架} \hfill 2026.04 -- 2026.05 \titlebreak
传统 chatbot 框架通常仅支持单一通道、固定工具集、流式与非流式路径割裂。初版MXWbot代码重复率40\%，工具调用失败后无降级机制，工具定义Token开销占输入50\%。基于\textbf{统一消息处理架构 + 三层降级保护 + 工具懒加载}重构。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{统一消息处理架构}：设计 \textbf{LoopPool + Loop八状态事件驱动引擎}（COMMAND → RESTORE → COMPACT → BUILD → RUN → SAVE → RESPOND → DONE），搭配 MessageBus 双向消息队列。Agent/Serve/Cron/Heartbeat 四种调用路径统一通过 Bus 投递，消除代码重复。Semaphore(20)全局并发控制 + per-session asyncio.Lock 串行化。
    \item \textbf{三层工具降级保护}：连续3次工具失败注入System提示强制LLM停止重试 → max\_iterations耗尽时追加一次无工具LLM调用产出自然语言回复 → 最终兜底文案。确保用户在任何极端情况下都能看到有意义的回复。
    \item \textbf{工具定义懒加载}：设计 GetToolSchema 元工具，首轮仅发送核心4个工具Schema，其余通过文本目录声明。LLM按需调用获取完整参数定义，同Session内缓存。工具数50+时可节省\textbf{65\%输入Token}。
    \item \textbf{双层记忆 + 多通道 + 安全防御}：短期记忆基于Session JSONL（Token裁剪pipeline），长期记忆基于SQLite+FTS5全文索引+异步LLM Summarizer。BaseChannel抽象层支持微信/QQ/Email三通道。SSRF校验、路径穿越防护、Token AES-GCM加密、命令注入检测等纵深防御。
\end{itemize}
