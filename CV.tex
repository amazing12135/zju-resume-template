\textbf{医疗 Agent 系统} \hfill 2026.02 -- 2026.04 \titlebreak
传统单 Agent 医疗问答系统存在\textbf{响应效率低、复杂问题分析不全面、多轮对话上下文理解能力弱}等问题。面对多症状、多维度的医疗咨询时，回答准确性与稳定性不足；同时缺乏长效记忆机制，无法连贯理解用户意图，难以满足实际医疗场景下高效、可靠、安全的智能问答需求。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{Skill-Agent两层架构}：底层构建7个原子Skills（知识检索、风险评估、症状分析、生活方式建议、ICD-10编码、临床指南、深度研究）供Agent使用，上层3个专业Agent（健康咨询、症状诊断、医学研究）根据任务需求自主选择调用，Skills和Agent完全解耦；
    \item \textbf{Agent loop执行机制}：基于ReAct框架实现Think-Act-Observe循环，LLM自主决策工具调用并观察结果，设置硬性限制最多调用2次工具防止过度调用，单Agent平均响应时间控制在15秒以内；
    \item \textbf{Agent Swarm}：智能路由判断问题复杂度（70\%单Agent快速通道/30\%Swarm协作模式），复杂问题自动分解任务并分配给多个Agent并行执行，Agent间通过共享上下文间接共享任务状态和执行结果；
    \item \textbf{双层记忆机制}：短期记忆管理会话级对话历史（单例模式确保所有Agent共享会话上下文，集成熵管理器自动去重压缩消息35\%，加载最近5轮对话），长期记忆通过Mem0云服务向量化存储会话总结支持跨会话相似案例检索；
    \item \textbf{Harness Engineering约束系统}:通过YAML文件显式定义Agent能力边界并运行时自动验证，实现输出自动修复机制（缺少免责声明自动添加、检测到高危症状自动添加就医警告）；
\end{itemize}
\textbf{项目成果：}                                                                                                                                  
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]                                                                                               
    \item \textbf{系统性能：}智能路由准确率从88\%提升至95\%，知识库检索准确率87\%，单Agent响应时间5-15秒，Swarm响应时间20-30秒，多轮对话上下文理解准确率从60\%提升至92\%；                                                                                                                       
    \item \textbf{回答质量：}医学专业人员三维度盲评打分，系统回答质量4.5分 vs doubao-seed-2.0-pro 3.9分；                                            
\end{itemize}

% --- 第二个项目 ---
\textbf{通用 AI Agent 框架} \hfill 2026.03 -- 2026.05 \titlebreak                                                                                                                 
传统单 Agent 聊天机器人框架普遍存在\textbf{多轮对话上下文膨胀严重、并发会话串行阻塞、工具调用安全边界模糊、流式响应卡顿}等问题。面对多用户高并发场景时，内存占用持续增长、LLM调用缺乏容错机制；同时缺乏系统级降级策略和 checkpoint 恢复能力，无法满足生产环境下高效、稳定、安全的多渠道智能 Agent 部署需求。                                                                                                                                                                  
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{8 状态事件驱动编排引擎}：基于显式状态转移表实现 \textbf{COMMAND→RESTORE→COMPACT→BUILD→RUN→SAVE→RESPOND→DONE} 流水线，Handler不直接路由、通过事件字符串驱动状态跃迁，斜杠命令短路跳过 LLM 调用阶段，RUN 异常自动回退至 RESTORE 恢复 checkpoint；
    \item \textbf{LoopPool 并发隔离架构}：全局 Semaphore(20) 控制并发上限 + 每会话 asyncio.Lock 串行化同一用户消息，长驻消费者从 Bus 输入队列轮询（1s 超时），stop() 1秒内优雅退出，统一消息处理路径（CLI/Cron/Heartbeat/Channel 均经 Bus 入队）；
    \item \textbf{ReAct Agent 执行引擎}：纯 ReAct 循环实现 LLM 调用→工具执行→结果注入的自主决策链，支持流式 delta 实时转发至Channel，写操作批量确认（每轮迭代单次确认弹窗），连续失败 3 次自动注入提示引导 LLM 停止重试，max_iterations 耗尽时触发最终无工具 LLM 调用生成文本回复；
    \item \textbf{三层记忆体系}：工作记忆（Session JSONL 持久化）、情节记忆（LLM 摘要压缩）、长期记忆（SQLite + FTS5 全文索引，重要性评分 + Jaccard 去重批量写入）；
    \item \textbf{Purpose 隔离上下文构建}：AGENT/SUBAGENT/SUMMARY/SYSTEM 四种调用目的隔离上下文组装策略，AGENT 获完整上下文（身份提示词 + 技能 XML + 会话摘要 + 长期记忆 + 工具目录 + 对话历史），SUMMARY 获最小上下文（零记忆递归依赖），延迟加载扩展工具 Schema 降低首轮 Token 消耗；
    \item \textbf{多渠道安全通信}：支持微信、QQ、邮件，统一确认 FIFO 队列 + TTL Task 生命周期管理；
    \item \textbf{6 维降级矩阵}：MCP 断连/内存库故障/Channel 断连/速率限制/磁盘满/Runner 异常 6 种故障场景预定义降级策略，checkpoint三重触发机制，Provider 4 次重试退避（1s→2s→4s）异常永不传播；
\end{itemize}

\textbf{项目成果：}
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{系统性能：}E2E 场景通过率 91.7\%（22/24），平均端到端延迟 6000+ms，纯文本问答 2000+ms，token 膨胀比目标 ≤ 1.3x；
    \item \textbf{稳定性与安全：}4 小时持续运行 0 崩溃，路径穿越 15 种变种拦截率 100\%，命令注入 10 种变种拦截率 100\%，checkpoint 恢复成功率 100\%；
    \item \textbf{工程完备性：}3 个 LLM Provider（OpenAI/Anthropic/DeepSeek）、3 个 Channel（微信/QQ/邮件）、10+原子工具（文件系统/Shell沙箱/Web搜索抓取/Cron调度），全异步协程架构支持 20 用户并发；
\end{itemize}
